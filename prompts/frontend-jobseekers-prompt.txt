# Antigravity Frontend Implementation Prompt – Module 3: Job Seekers / Users

## Objective

Implement the complete **frontend** for the **Job Seekers / Users Module** in Antigravity using **React** and **Tailwind CSS**.

This phase focuses **only on the frontend**. The backend APIs, authentication, business logic, database operations, validations, and file storage have already been implemented.

The frontend should consume the existing backend APIs and provide a modern, responsive, intuitive, and production-ready user experience.

The **Admin** and **Recruiter** modules have already been completed. The Job Seeker module must seamlessly integrate with the existing application, following the same design language, architecture, and user experience standards while providing a personalized experience tailored to each candidate.

---

# Technology Stack

Use:

* React
* Tailwind CSS
* React Router
* Axios (or the existing API service)
* React Hook Form (preferred for forms)
* Appropriate state management already used in the project (Context API/Redux/Zustand if applicable)

Maintain consistency with the existing Antigravity design system, spacing, typography, reusable components, and color palette.

---

# Authentication Flow

The Job Seeker must use the **same Login and Sign Up page** already implemented in Antigravity.

Do not create a separate authentication page.

After successful authentication:

### First-Time Login

If the backend indicates that the candidate profile does not exist, automatically redirect the user to the Candidate Profile Wizard.

---

### Returning User

If the backend indicates that the candidate profile already exists, redirect the user directly to their Portfolio Dashboard.

# Personalized Candidate Experience

Unlike the Admin and Recruiter modules, the Candidate interface should provide a personalized experience based on the candidate's own profile.

As profile information is completed, the interface should dynamically adapt using the candidate's data.

Examples include:

* Display the candidate's profile photo throughout the application.
* Display the candidate's full name and professional headline prominently.
* Personalize dashboard greetings using the candidate's information.
* Show profile completion percentage.
* Highlight completed sections while encouraging completion of incomplete sections.
* Present Education, Experience, Projects, Certifications, Languages, and Skills as a professional resume-style portfolio rather than simple form data.
* Instantly reflect profile updates throughout the interface.
* Clearly associate uploaded certification documents with their respective certification entries.
* Allow candidates to easily view and download uploaded certificates.
* Make the portfolio feel like the candidate's own professional website and digital identity.

The overall experience should give candidates the feeling of building and maintaining their own premium professional portfolio.
